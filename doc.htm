User Manual

Introduction

The User Manual will guide you through installing and playing the game. This guide will focus on Windows and Ubuntu-specific commands. If you are using a different UNIX operating system, please refer to your OS manual for specific commands.

Installation

Traverse to the desired installation directory. In this example, a new directory called Games will be created.

cd ~                 # traverse to the home directory
mkdir Games          # create Games directory
cd Games             # traverse into Games directory

64-bit Windows:

Download the executable from here:
https://github.com/downloads/ThrosturX/Alpha/TicTacToe.exe

64-bit Ubuntu

Download the binary file from here:
https://github.com/downloads/ThrosturX/Alpha/TicTacToe

Other operating systems:

Download the installation files from github:

https://github.com/ThrosturX/Alpha/archive/master.zip

This can be done using the command line on UNIX

wget   https://github.com/ThrosturX/Alpha/archive/master.zip
unzip master.zip     # extracts the contents
cd Alpha-master      # change into the new directory

The game has already been compiled into 64-bit binaries for Windows and Ubuntu, if you are using one of these, you can skip this step. For other systems, game must be compiled. This can be done using make, which is included with most UNIX systems. G++ is also required to compile the code manually and must be installed by an Administrator. The rule that is used to correctly deploy the game is called install. This will install the game in the ./release/ directory.

sudo apt-get install –y g++    # install g++
make install                       # compile sources

 Some users may want to get rid of the sources and keep the game in a different directory. In this example, a new directory called TicTacToe will be created in the Games directory. The game will then be copied to the TicTacToe directory and the sources will be removed. In a windows environment, this can all be easily done using Windows Explorer.

mkdir ../TicTacToe   # create TicTacToe directory
cp release/TicTacToe ../TicTacToe/TicTacToe # copy game
cd ..                # traverse back to Games
rm –rf Alpha              # remove sources

Playing Tic Tac Toe

On 64-bit Windows or Ubuntu, simply run the downloaded executable or binary file.

On UNIX, traverse into the directory in which the game was installed (release by default). The game can be run by running it normally:

./TicTacToe

The game may need permission to run. It can be granted permission with the following command:

chmod +x TicTacToe

Tic Tac Toe is a two player game. This game does not support single-player mode. The player who moves first is called ‘X’ whereas the player who moves second is called ‘O’. The objective of the game is for a player to get three of his symbols to line up in a row. The first player begins by selecting a tile on the board to place his symbol, the second player the does the same. The players continue taking turns until the game ends, at which point a scoreboard will be printed, showing the current scores of the players.

Tiles can be selected by entering input in the format “C N” where C is A,B or C and N is 1, 2 or 3. When the game is completed, it can be played again by typing any word starting with ‘y’. Instead of giving input for a tile, the commands “reset” and “exit” also work and do exactly as they seem to.

 

Administration Manual

Introduction

The Administration Manual will guide you through downloading the game’s sources for development and testing. This guide will focus on Ubuntu-specific commands. If you are using a different operating system, please refer to your OS manual for specific commands.

For instructions on how to install the program and play the game, please refer to the User Manual.

Installation

Ensure that you have G++, CPPCheck and Git installed before continuing. On Ubuntu, these programs can be downloaded using the package manager on the command line:

sudo apt-get install –y g++
sudo apt-get install –y git
sudo apt-get install –y cppcheck

Once G++, CPPCheck and Git has been installed, traverse to the desired development directory. In this example, a new directory called Dev will be created.

cd ~                 # traverse to the home directory
mkdir dev            # create dev directory
cd dev               # traverse into Games directory

Now clone the Git repository from Github. The repository is located at https://github.com/ThrosturX/Alpha.git.

git clone https://github.com/ThrosturX/Alpha.git

The repository includes UnitTest++ which should work on UNIX operating systems with the Make file. The Make file includes different rules, specifically tailored to specific tasks. Please note that the Make file may report errors such as “cannot create directory” if the directory already exists – this kind of error can be ignored as the Make file will use the existing directory instead. Below is a brief list of the more common rules and what they are for:

install:    # builds installs the game into the ./release/ directory.

deploy:         # updates the repository, builds and runs all tests, builds and installs the game into the ./release/ directory

checkin:    # builds and runs unit tests, commits changes and attempts to push into the repository if tests succeed

build:      # builds and runs unit tests

e2e:        # runs end-to-end tests

test-build: # builds and runs all tests in a format friendly for outputting results to file

Continuous Integration

To configure your build server to automatically send emails on failed tests, edit the emails file to include the email addresses of those who should receive failure emails. A failure email will include a log file of the failed build which should indicate the errors and assist with debugging.

Once you have added these email addresses, install and set up mutt with a valid email address to ensure that attachments can be sent.

sudo apt-get install –y mutt

To run timely builds, type the following command to enter cronjob:

crontab –e

 Each cronjob has following syntax:

# +---------------- minute (0 - 59)
# |  +------------- hour (0 - 23)
# |  |  +---------- day of month (1 - 31)
# |  |  |  +------- month (1 - 12)
# |  |  |  |  +---- day of week (0 - 6) (Sunday=0 or 7)
# |  |  |  |  |
  *  *  *  *  *  command to be executed

For example, to run the build script every 10 minutes, type in the following:

*/10 * * * * /[dir]/test-build.sh

In this case, [dir] should be the full path to the repository, such as /home/user/dev/Alpha/.
